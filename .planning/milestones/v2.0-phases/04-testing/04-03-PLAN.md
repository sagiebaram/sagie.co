<task type="auto">
  <name>Task 1: E2E form submission tests</name>
  <files>tests/forms.spec.ts</files>
  <action>
Create `tests/forms.spec.ts` with E2E tests for all 4 application forms.

**Strategy per user decision:** Mock API responses via `page.route()` — intercept POST to `/api/applications/*` and return `{ success: true }`. No real Notion writes.

**Before writing tests:** Read each form component to discover:
- The actual page URL (e.g., /apply, /apply/chapter, /apply/solutions, /apply/ventures)
- The field names used in `name` attributes on form inputs
- The submit button selector
- The success message text/selector after submission

Find the form components:
```bash
find src -name "*Form*" -o -name "*form*" | grep -i member
find src -name "*Form*" -o -name "*form*" | grep -i chapter
```

For each form (membership, chapter, solutions, ventures):
1. Register `page.route('**/api/applications/{formName}', ...)` BEFORE `page.goto()`
   - Match method === 'POST' -> fulfill with `{ json: { success: true }, status: 200 }`
   - Otherwise `route.continue()`
2. Navigate to the form page
3. Fill all required fields using `page.fill('[name="fieldName"]', 'value')` or equivalent locators
   - If fields use custom input components without standard name attributes, use `page.getByLabel()` or `page.getByPlaceholder()` instead
4. Click submit
5. Assert success state is visible: `await expect(page.getByText(/thank you|success|submitted/i)).toBeVisible({ timeout: 10000 })`

IMPORTANT:
- The `page.route()` glob pattern must use `**` prefix to match any host: `**/api/applications/membership`
- Read the actual form components to get the correct field selectors — do NOT guess
- Forms may use hidden fields for honeypot (_trap) and timing (_t) — the route mock bypasses server validation, so these don't matter for E2E
- If a form has a multi-step flow or dropdown selects, handle those interactions appropriately
- Set reasonable timeout on success assertion (forms may have transition animations)
