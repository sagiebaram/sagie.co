<objective>
Fix form submission failures caused by empty optional URL fields and invisible 422 error messages.

Purpose: All optional URL fields (linkedIn, website, pitchDeckUrl, portfolioUrl, url) use `.url().optional()` which rejects empty strings (Zod optional only accepts undefined, not ''). When a user leaves LinkedIn blank, the browser sends '', Zod rejects it client-side via zodResolver, but the error is barely visible. On the server side, if somehow reached, the 422 response body with fieldErrors is ignored by the client.

Output: Schemas transform empty strings to undefined before URL validation. All form components parse 422 fieldErrors and display them visibly.
</objective>

  <action>
In all 6 form components, update the `!res.ok` error handling block in the onSubmit function. Currently most forms either do `setSubmitError('Something went wrong.')` or silently return.

Replace the `if (!res.ok)` block in each form with:

```typescript
if (!res.ok) {
  try {
    const body = await res.json()
    if (body.fieldErrors && typeof body.fieldErrors === 'object') {
      // Server returned per-field validation errors (422)
      const firstError = Object.values(body.fieldErrors).flat()[0]
      setSubmitError(typeof firstError === 'string' ? firstError : 'Please check your input and try again.')
    } else if (body.error && typeof body.error === 'string') {
      setSubmitError(body.error)
    } else {
      setSubmitError('Something went wrong. Please try again.')
    }
  } catch {
    setSubmitError('Something went wrong. Please try again.')
  }
  return
}
```

For forms that currently don't have setSubmitError (ChapterForm, SolutionsForm, SuggestEventForm):

{submitError && (
