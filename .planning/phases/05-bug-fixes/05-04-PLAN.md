  <action>
In the filterKey useEffect (lines 60-82), after the isFirstRender check, change the logic to:

1. Keep the existing container fade (el.style.opacity = '0' then rAF to '1') -- this handles the overall container.

2. ALSO target child elements when a selector is provided. After the container opacity is set to '1' inside the rAF callback, use gsap.utils.toArray to find all child elements matching the selector within el, then set each child's inline style.opacity to '1'. This overrides the CSS opacity:0 from globals.css.

The updated filterKey useEffect should look like:

```typescript
useEffect(() => {
  if (filterKey === undefined) return

  if (isFirstRender.current) {
    isFirstRender.current = false
    return
  }

  const el = ref.current
  if (!el) return

  el.style.opacity = '0'
  el.style.transition = ''

  const rafId = requestAnimationFrame(() => {
    el.style.transition = 'opacity 200ms ease-out'
    el.style.opacity = '1'

    // Override CSS opacity:0 on child card elements
    if (selector) {
      const children = gsap.utils.toArray<HTMLElement>(selector, el)
      children.forEach((child) => {
        child.style.opacity = '1'
      })
    }
  })

  return () => {
    cancelAnimationFrame(rafId)
  }
}, [filterKey, selector])
```

Note: `selector` must be added to the dependency array since it is now used inside the effect. This is safe because selector is a stable string prop that does not change between renders.

Do NOT modify globals.css -- the opacity:0 rules are correct for initial page load (GSAP handles first render). The fix is that subsequent filter changes need inline opacity override on newly rendered children.

Do NOT modify the useLayoutEffect (GSAP scroll-reveal animation) -- it is correct for initial mount.
