Create `proxy.ts` at project root (Next.js 16 convention — replaces middleware.ts).

The proxy function must:
1. Generate a per-request nonce using `Buffer.from(crypto.randomUUID()).toString('base64')`
2. For `/api/*` routes: check the `origin` header against `ALLOWED_ORIGINS` env var. If origin is present AND not in allowed set, return 403 JSON `{ error: 'Forbidden' }`. If origin is absent (same-origin/server-to-server), allow the request through.
3. Build CSP header as an array joined by '; ':
   - `default-src 'self'`
   - `script-src 'self' 'nonce-${nonce}' 'strict-dynamic'` (add `'unsafe-eval'` only in dev)
   - `style-src 'self' 'unsafe-inline'` (unchanged — nonces on styles not in scope)
   - `img-src 'self' data: https:`
   - `font-src 'self' https: data:`
   - `object-src 'none'`
   - `base-uri 'self'`
   - `form-action 'self'`
   - `frame-ancestors 'none'`
   - `connect-src 'self' https://api.notion.com https://*.ingest.sentry.io`
4. Set CSP on both request headers and response headers
5. Set `x-nonce` request header so downstream components can read it via `headers()`
6. Export `config` with matcher excluding static files, prefetches

In `next.config.ts`: Remove the `script-src 'self' 'unsafe-inline'` line from the `csp` array entirely. proxy.ts now owns script-src. Keep all other security headers (X-Frame-Options, X-Content-Type-Options, Referrer-Policy, Permissions-Policy) in next.config.ts — those are static. Also remove the `Content-Security-Policy` header from the headers() return entirely since proxy.ts will set the full CSP. Keep the non-CSP headers and the `/api/:path*` Cache-Control header.

Use `export function proxy()` (not `middleware`) — this is the Next.js 16 convention.

**Rate limiting in validation.ts:**

Add above the `withValidation` function:
1. Type: `type RateEntry = { count: number; firstHit: number }`
2. Module-level store: `const rateStore = new Map<string, RateEntry>()`
3. Constants: `const RATE_LIMIT = 5` and `const WINDOW_MS = 10 * 60 * 1000` (10 minutes)
4. Helper `getIP(req: Request): string` — reads `x-forwarded-for` header, splits on comma, takes first entry, trims. Falls back to `'unknown'`.
5. Helper `isRateLimited(ip: string): { limited: boolean; retryAfter: number }`:
   - If no entry or entry expired (now - firstHit > WINDOW_MS), create new entry with count 1, return not limited
   - If entry.count >= RATE_LIMIT, compute retryAfter in seconds, return limited
   - Otherwise increment count, return not limited

Inside `withValidation`, add rate limit check BEFORE the honeypot check (so rate-limited IPs get 429 even if they include honeypot):
```typescript
const ip = getIP(req)
const { limited, retryAfter } = isRateLimited(ip)
if (limited) {
  return NextResponse.json(
    { error: "You've submitted several times recently. Please wait a few minutes before trying again." },
    { status: 429, headers: { 'Retry-After': String(retryAfter) } }
  )
}
```

The friendly message text is a locked user decision — use it exactly.
