**SubmitResourceForm (DIFFERENT PATTERN — per Pitfall 5):**
- This form does NOT use FormField component — it uses plain `<input>` elements with a horizontal inline layout (`flex-col sm:flex-row`).
- Schema: ResourceSchema (defined inline in route.ts, not in schemas.ts): `{ name: z.string().min(1).max(200).trim(), url: z.string().url().max(500).trim() }`
- API endpoint: `/api/submit-resource`
- This form has NO honeypot/timing protection (it didn't have it before — it's a simple 2-field sidebar widget)

**SubmitResourceForm migration approach:**
1. Import `useForm` and `zodResolver`. Define a local schema or import from route:
   ```typescript
   import { useForm } from 'react-hook-form'
   import { zodResolver } from '@hookform/resolvers/zod'
   import { z } from 'zod'

   const ResourceSchema = z.object({
     name: z.string().min(1, 'Give it a name').max(200).trim(),
     url: z.string().url('Please enter a valid URL').max(500).trim(),
   })
   type FormData = z.infer<typeof ResourceSchema>
   ```
   (Define schema locally in the component — don't import from route.ts since that's a server file)

2. Setup: `useForm<FormData>({ resolver: zodResolver(ResourceSchema), mode: 'onBlur', reValidateMode: 'onChange' })`

3. Apply `{...register('name')}` and `{...register('url')}` directly to the existing `<input>` elements. Remove `value` and `onChange` props. Keep ALL existing inline styles, className, onFocus/onBlur border color changes.

4. IMPORTANT: The onFocus/onBlur border handlers need to coexist with RHF's onBlur (for validation). RHF's `register()` returns an `onBlur`. Merge them:
   ```typescript
   const nameReg = register('name')
   // On the input:
   <input
     {...nameReg}
     onBlur={(e) => { nameReg.onBlur(e); e.currentTarget.style.borderColor = 'var(--border-default)' }}
     onFocus={(e) => { e.currentTarget.style.borderColor = 'var(--silver)' }}
     // ... rest of props
   />
   ```

5. Keep submitWarning + rateLimitUntil state (Phase 5 rate limit handling).
6. Remove `name`/`url` useState — replaced by RHF.
7. Keep success state for success message display.
8. After successful submit, call `reset()` from useForm to clear fields (replaces `setName('')`/`setUrl('')`).
9. Show error messages below each input for blur validation:
   ```typescript
   {errors.name && <span style={{ fontSize: '10px', color: '#c0392b' }}>{errors.name.message}</span>}
   ```
10. PRESERVE the existing horizontal layout — `flex flex-col sm:flex-row gap-4`. Do NOT wrap in FormField components.
11. The form already has a `<form onSubmit={handleSubmit}>` — update to `<form onSubmit={handleSubmit(onSubmit)}>`.
