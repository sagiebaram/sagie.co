Replace the `type === 'select'` branch with a custom-styled dropdown. Implementation requirements:

1. **Trigger button:** Styled identically to current input (`inputStyle` base), text-align left, a subtle chevron on the right (CSS border-triangle or Unicode `\u25BE`), displays selected value or placeholder text. Add `role="combobox"`, `aria-expanded`, `aria-haspopup="listbox"`. Add `data-dropdown={name}` attribute for test selectors. Add `tabIndex={0}` for keyboard focus.

Read tests/forms.spec.ts fully first.

The current tests use `page.selectOption('[name="role"]', 'Founder')` etc. which only works with native `<select>`. With the custom dropdown built in Task 1, the interaction pattern changes to:

1. Click the dropdown trigger (identified by `data-dropdown` attribute)
2. Wait for the options panel to appear
3. Click the desired option text

Create a reusable helper function at the top of the test file:

```typescript
async function selectDropdownOption(
  page: import('@playwright/test').Page,
  fieldName: string,
  optionText: string
) {
  // Click the dropdown trigger
  await page.locator(`[data-dropdown="${fieldName}"]`).click()
  // Click the option in the listbox
  await page.getByRole('option', { name: optionText }).click()
}
```

1. **Membership form submission test:** Replace:
