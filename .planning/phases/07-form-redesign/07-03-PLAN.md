**Part B: Replace native `<select>` with custom dropdown**

Replace the `type === 'select'` branch in FormField with a custom-styled dropdown. The dropdown must:

1. **Trigger button:** Styled identically to the current input (same `inputStyle` base), with text-align left, a subtle chevron on the right (CSS triangle or Unicode arrow), displays the selected value or "Select..." placeholder. The trigger needs `role="combobox"`, `aria-expanded`, `aria-haspopup="listbox"`.

2. **Options panel:** Absolutely positioned below trigger, same background as `var(--bg-card)`, border `0.5px solid var(--border-default)`, z-index 50. Each option styled with padding, hover state using `var(--border-default)` background. The panel has `role="listbox"`.

3. **Selection behavior:**
   - Clicking trigger toggles open/close
   - Clicking an option selects it, closes the panel
   - Selected option highlighted with `var(--silver)` text color
   - On selection: call `onChange(value)` if using legacy mode, or for RHF integration use a hidden input approach

4. **RHF integration approach:** The custom dropdown cannot use `register()` directly (register returns refs for native inputs). Instead:
   - Add a new optional prop: `onValueChange?: (value: string) => void` to FormFieldProps
   - In the form components, use `setValue` and `watch` from useForm instead of `register` for select fields:
     ```typescript
     const { register, handleSubmit, setValue, watch, formState: { errors } } = useForm(...)
     const roleValue = watch('role')
     // In JSX:
     <FormField
       type="select"
       name="role"
       value={roleValue || ''}
       onValueChange={(v) => setValue('role', v, { shouldValidate: true })}
       error={errors.role?.message}
       ...
     />
     ```
   - This means for select-type fields, forms use `value` + `onValueChange` instead of `registration`. The `registration` prop is still used for text/email/textarea/url fields.
   - Add `{ shouldValidate: true }` to `setValue` so onBlur-equivalent validation fires on selection.

5. **Click-outside handling:** Use a `useRef` for the dropdown container and a `useEffect` with a `mousedown` listener on `document`. If click target is outside the ref, close the panel. Clean up the listener on unmount.

6. **Keyboard navigation:**
   - When trigger is focused and panel is closed: Enter/Space opens panel, arrow down opens and highlights first option
   - When panel is open: Arrow up/down moves highlight, Enter selects highlighted option and closes, Escape closes without selecting
   - Use `onKeyDown` handler on the trigger and the listbox

7. **Styling tokens:** Use existing CSS custom properties:
   - Background: `var(--bg-card)`
   - Border: `var(--border-default)`
   - Text: `var(--text-primary)` for selected, `var(--text-secondary)` for options
   - Hover: `var(--border-default)` as background
   - Selected indicator: `var(--silver)` text color
   - Font: `var(--font-body)`, 13px, matching inputStyle

8. **"Other" + free-text support:** Add an optional prop `allowOther?: boolean` to FormFieldProps. When `allowOther` is true and the user selects "Other":
   - The dropdown closes
   - A text input appears below the dropdown trigger (same styling as regular text inputs)
   - The text input's value becomes the field value (via `onValueChange`)
   - If user re-opens dropdown and picks a predefined option, the text input disappears
   - The "Other" option must be included in the `options` array by the consuming form
