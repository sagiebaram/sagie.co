  <files>package.json, package-lock.json, src/components/ui/FormField.tsx</files>
  <action>
1. Install packages:
   ```bash
   npm install react-hook-form@^7.72.0 @hookform/resolvers@^5.2.2 --legacy-peer-deps
   ```
   (--legacy-peer-deps required per Phase 5 decision: @eslint/js@10 vs eslint@9 peer conflict)

2. Update FormField component to support both old (value/onChange) and new (registration) prop interfaces, plus checkbox-group type:

   - Import `UseFormRegister` type from `react-hook-form` for the registration prop type
   - Update `FormFieldProps` interface:
     - Change `type` union to include `'checkbox-group'`
     - Add `registration?: UseFormRegisterReturn` prop (from `react-hook-form`)
     - Make `value` and `onChange` optional (backward compat during migration, but new forms use registration only)
   - For `input`, `textarea`, and `select` elements:
     - If `registration` is provided, spread `{...registration}` onto the element (this provides name, ref, onChange, onBlur from RHF)
     - If `registration` is NOT provided, use existing `value`/`onChange` pattern (backward compat)
     - Remove the explicit `name={name}` and `onChange` when registration is spread (registration includes these)
   - Add `checkbox-group` rendering branch (when `type === 'checkbox-group'`):
     - Render a flex-wrap container with gap: 10px
     - For each option in `options`, render a `<label>` with:
       - A custom-styled checkbox input: `appearance: none`, width/height 16px, border `0.5px solid var(--border-default)`, background `var(--bg-card)`, cursor pointer
       - Checked state via CSS: add a `<style>` tag or use inline approach — on `:checked`, set background to `var(--silver)` with a subtle inner shadow or border change. Since inline styles can't handle `:checked`, use a className approach: add a `<style jsx>` or a global CSS class `.form-checkbox:checked { background: var(--silver); border-color: var(--silver); }` — OR use a controlled approach where the checkbox native check indicator with filter/accent-color works. Simplest: use `accentColor: 'var(--silver)'` and keep `appearance: auto` but override sizing, OR keep `appearance: none` and use a pseudo-element approach via a tiny inline style tag in the component.
       - RECOMMENDED approach: Keep `appearance: none` for the checkbox. Add a scoped `<style>` block at the top of the checkbox-group section that targets `.rhf-checkbox:checked` to set `background: var(--silver); border-color: var(--silver);`. Apply className `rhf-checkbox` to each checkbox input alongside inline styles.
       - Label text: fontSize 11px, letterSpacing 0.08em, color var(--text-secondary), fontFamily var(--font-body)
     - Each checkbox input gets `{...registration}` spread (same name for all = RHF collects into array) plus `value={opt}`
   - Error display stays the same (below the input group)

3. Verify FormField compiles: `npx tsc --noEmit src/components/ui/FormField.tsx` (or just `npm run build` to check for type errors)
