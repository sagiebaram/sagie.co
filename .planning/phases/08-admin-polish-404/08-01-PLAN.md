  <action>
Refactor `src/app/admin/revalidate/page.tsx` — replace the single `loading` boolean and `result` string with a per-button status architecture:

**State changes:**
1. Remove `loading` and `result` state variables entirely
2. Add `type ButtonStatus = 'idle' | 'loading' | 'success' | 'error'`
3. Add `const [statuses, setStatuses] = useState<Map<string, ButtonStatus>>(new Map())`
4. Add helper functions `setStatus(key, status)` (always `new Map(prev).set(key, status)` — never mutate in place) and `getStatus(key)` (returns `statuses.get(key) ?? 'idle'`)
5. Add `const ALL_KEY = '__all__'` sentinel for the "Refresh All" button
6. Add `const [showSecretHint, setShowSecretHint] = useState(false)` for 401 hint message
7. Add `const timersRef = useRef<Map<string, ReturnType<typeof setTimeout>>>(new Map())` — import `useRef` from React

**Timer management:**
1. Add `scheduleReset(key, delay, resetFn)` — clears any existing timer for the key first, then sets new setTimeout, stores in timersRef
2. Add cleanup `useEffect` that on unmount clears all timers from timersRef
3. On rapid clicks: existing timer for that key is cleared before new one is set (prevents timer accumulation)

**handleRevalidate refactor:**
1. Change signature to `handleRevalidate(key: string, tags: string[])` — key identifies which button
2. Set `setStatus(key, 'loading')`
3. On `res.status === 401`: set status to 'error', then `scheduleReset(key, 2000, () => resetToPrompt(true))` — the `true` indicates 401 caused the reset
4. On other `!res.ok`: set status to 'error', then `scheduleReset(key, 3000, () => setStatus(key, 'idle'))`
5. On success: set status to 'success', then `scheduleReset(key, 3000, () => setStatus(key, 'idle'))`
6. On catch: same as other error path
7. For "Refresh All": call `handleRevalidate(ALL_KEY, [])` — the ALL_KEY gets its own independent status

**resetToPrompt function:**
1. Accept `wasUnauthorized: boolean` parameter
2. `setSecretEntered(false)`, `setSecret('')`, `setStatuses(new Map())`
3. If `wasUnauthorized`: `setShowSecretHint(true)`
4. Clear ALL pending timers: `timersRef.current.forEach(id => clearTimeout(id))`, then `timersRef.current.clear()`

**Inline SVG icon components** (define inside the file, above the default export):
1. `Spinner()` — 16x16 SVG circle with partial arc, `style={{ animation: 'icon-spin 1s linear infinite' }}`, `aria-hidden="true"`
2. `CheckIcon()` — 16x16 SVG checkmark path in `#4ade80` (green-400), `aria-hidden="true"`
3. `XIcon()` — 16x16 SVG X path in `#f87171` (red-400), `aria-hidden="true"`
4. `ButtonContent({ status, label })` — renders Spinner/CheckIcon/XIcon/label based on status

**Secret prompt updates:**
1. When `showSecretHint` is true, show `<p className="text-amber-400 text-xs mb-3">Secret was invalid or has been rotated. Please re-enter.</p>` above the input
2. On successful secret entry (Continue click or Enter), set `setShowSecretHint(false)` to clear the hint

**Button styling — apply Button.tsx variant classes to native button elements:**
1. Individual tag buttons: apply outline variant classes — `border border-silver text-silver hover:bg-silver hover:text-background text-button tracking-button px-[34px] py-4 font-body uppercase transition-all duration-150 hover:-translate-y-px` + `disabled:opacity-50 disabled:cursor-not-allowed`
2. "Refresh All" button: apply primary variant classes — `bg-white [color:black] hover:opacity-85 text-button tracking-button px-[34px] py-4 font-body uppercase transition-all duration-150 hover:-translate-y-px` + `disabled:opacity-50 disabled:cursor-not-allowed`
3. Each button: `disabled={getStatus(buttonKey) === 'loading'}` — only disabled when ITS OWN request is in flight, NOT globally
4. "Change secret" link: keep as text button or use ghost variant styling
5. Secret prompt "Continue" button: apply primary variant styling for consistency
6. Keep the gray-950 dark admin aesthetic for page/card backgrounds — brand styling is buttons/typography only

**Dashboard button rendering:**
1. Each `CONTENT_TYPES.map(({ label, tag }) => ...)` button: `onClick={() => handleRevalidate(tag, [tag])}`, renders `<ButtonContent status={getStatus(tag)} label={label} />`
2. "Refresh All" button: `onClick={() => handleRevalidate(ALL_KEY, [])}`, renders `<ButtonContent status={getStatus(ALL_KEY)} label="Refresh All" />`
3. Remove the old `{result && ...}` text block entirely — feedback is now inline per-button
