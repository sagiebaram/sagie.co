  <action>
1. Create src/app/api/revalidate/route.ts:
   - Import `revalidateTag` from 'next/cache' and `env` from '@/env/server'
   - Define ALL_TAGS constant: `['notion:blog', 'notion:events', 'notion:resources', 'notion:solutions', 'notion:members', 'notion:chapters']`
   - Export async POST handler (do NOT use withValidation — this endpoint has its own auth via secret, not form validation):
     a. Parse JSON body: `const body = await request.json()`
     b. Check secret: `if (body.secret !== env.REVALIDATE_SECRET) return Response.json({ error: 'Unauthorized' }, { status: 401 })`
     c. If REVALIDATE_SECRET is undefined (optional env var), reject ALL requests with 401 — endpoint is disabled until secret is configured
     d. Determine tags: `const tags: string[] = body.tags?.length ? body.tags : ALL_TAGS`
     e. Loop: `for (const tag of tags) { revalidateTag(tag, 'max') }` — MUST use two-arg form, single-arg is deprecated in Next.js 16
     f. Return: `Response.json({ revalidated: true, tags, now: Date.now() })`
   - Wrap in try/catch: malformed JSON returns 400

2. Create src/app/admin/revalidate/page.tsx:
   - Add `'use client'` directive — this is a client component (user interaction, state)
   - Simple functional admin tool page (not marketing page)
   - State: `secret` (string), `loading` (boolean), `result` (string | null)
   - UI flow:
     a. Password prompt: text input for REVALIDATE_SECRET (type="password"), Submit button
     b. Once secret entered, show grid of revalidation buttons:
        - One button per content type: Blog, Events, Resources, Solutions, Members, Chapters
        - One "Refresh All" button
     c. Each button calls `fetch('/api/revalidate', { method: 'POST', body: JSON.stringify({ secret, tags: [tagName] }) })`
     d. "Refresh All" sends `{ secret, tags: [] }` (empty = flush all)
     e. Show result text below buttons: "Revalidated: notion:blog" or error message
   - Styling: minimal Tailwind, dark background matching site theme, functional layout
   - The secret is typed by the user at runtime — NEVER read from process.env or env on client side
   - No layout.tsx needed — inherits from root layout
   - Add metadata export: `export const metadata = { title: 'Admin - Revalidate', robots: 'noindex' }`

   WAIT: metadata export is not allowed in client components. Either:
   - Option A: Create a separate layout.tsx in src/app/admin/revalidate/ with the metadata export
   - Option B: Skip metadata, add a robots meta tag via next/head or just accept it

   Use Option A: Create src/app/admin/revalidate/layout.tsx as a server component with metadata export `{ title: 'Admin - Revalidate', robots: { index: false } }` that renders children. Keep page.tsx as 'use client'.
